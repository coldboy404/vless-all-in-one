# 使用说明（精简版）

本页只保留维护者视角下真正有用的说明，不再复制上游的大篇幅面向终端用户教程。

## 基本使用方式

通常通过以下方式运行脚本：

```bash
chmod +x vless-server.sh
./vless-server.sh
```

如果脚本安装了快捷命令，也可能通过 `vless` 进入菜单。

## 建议的使用顺序

1. 先审查脚本
2. 再在测试机执行
3. 确认端口、防火墙、证书策略
4. 再决定是否用于正式环境

## 推荐优先确认的点

### 系统兼容
根据上游说明，脚本面向：
- Alpine
- Debian
- Ubuntu
- CentOS

但不同发行版分支逻辑不一定完全一致，实际部署前应逐条验证。

### 网络侧影响
脚本可能改动：
- 监听端口
- Nginx / Xray / Sing-box 配置
- iptables 或相关转发规则
- 证书与域名解析相关设置

### 配置持久化
从脚本头部可见，配置目录默认涉及：
- `/etc/vless-reality`

建议先确认：
- 配置写入结构
- 数据库 JSON 结构
- 升级时兼容性
- 卸载是否完整回收

## 不建议直接做的事

- 不审计脚本就直接上生产
- 不做快照就直接覆盖现有代理环境
- 在未知端口策略下直接与现网服务混用
- 未确认依赖版本就直接做自动更新
